## 🚀 如何访问这些数据

### API接口

#### 1. 查看待审核知识列表
```bash
GET /api/enterprise-knowledge/mining/pending-review?page=1&limit=10
```

**返回示例**:
- 5条待审核记录
- 包含AI评分、置信度、挖掘理由等信息

#### 2. 查看负反馈列表
```bash
GET /api/enterprise-knowledge/feedback/list?page=1&limit=10
```

**返回示例**:
- 7条反馈记录 (4条未处理 + 3条已处理)
- 包含关联的知识库详情

## ⚠️ 注意事项

1. **负反馈计数已同步**: 相关知识库的 `negative_feedback_count` 字段已自动更新

2. **用户ID统一**: 所有反馈记录的 `user_id` 都是 1 (admin用户)

3. **数据可重置**: 如需重新导入，先删除再执行SQL即可:
   ```sql
   DELETE FROM knowledge_pending_review WHERE id > 0;
   DELETE FROM knowledge_feedback WHERE id > 0;
   ```

4. **前端页面**: 刷新浏览器后即可看到新数据，无需重启服务
